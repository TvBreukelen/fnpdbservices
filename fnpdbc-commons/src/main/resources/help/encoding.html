Characterset Encoding

The term "characterset" refers to how the characters of what has been typed in on the screen are stored in a file. In this file each character will have a unique binary value, defined by the characterset.
It is very important for programs like DBConvert and FNProg2PDA to find out which characterset is used to ensure that the information stored on disk is interpreted correctly (encoded).

DBConvert/FNProg2PDA and Characterset Encoding

If your file to be exported contains non-Latin characters (like Greek, Turkish or Chinese), you may have encountered the problem that your export file contains strange values (like questionmarks). Since files are not marked with a signature to denote the encoding used, DBConvert/FNProg2PDA will always export files using the default encoding, unless you override the default by selecting an alternative characterset in this screen as default.

Notes: Every instance of DBConvert/FNProg2PDA has a default characterset, which may or may not be one of the standard charactersets. The default characterset is determined during startup and typically depends upon the locale and characterset being used by the underlying operating system.

Most western european languages fit into the characterset ISO-8859-1 (Latin-1). Other languages, like Greek, Polish, Chinese or Japanese have more complicated encodings (see below).

Standard Encodings

  • European languages
    ASCII, ISO-8859-{1,2,3,4,5,7,9,10,13,14,15,16}, KOI8-R, KOI8-U, KOI8-RU, CP{1250,1251,1252,1253, 1254,1257}, CP{850,866}, Mac{Roman,CentralEurope,Iceland,Croatian,Romania}, Mac{Cyrillic,Ukraine, Greek,Turkish}, Macintosh
  • Semitic languages
    ISO-8859-{6,8}, CP{1255,1256}, CP862, Mac{Hebrew,Arabic}
  • Japanese
    EUC-JP, SHIFT_JIS, CP932, ISO-2022-JP, ISO-2022-JP-3, ISO-2022-JP-2, ISO-2022-JP-1, EUC-JISX0213, Shift_JISX0213
  • Chinese
    EUC-CN, HZ, GBK, GB18030, EUC-TW, BIG5, CP950, BIG5-HKSCS, ISO-2022-CN, ISO-2022-CN-EXT
  • Korean
    EUC-KR, CP949, ISO-2022-KR, JOHAB
  • Tajik
    KOI8-T
  • Thai
    TIS-620, CP874, MacThai
  • Laotian
    MuleLao-1, CP1133
  • Vietnamese
    VISCII, TCVN, CP1258
  • Full Unicode
    UTF-8
    UCS-2, UCS-2BE, UCS-2LE
    UCS-4, UCS-4BE, UCS-4LE
    UTF-16, UTF-16BE, UTF-16LE
    UTF-32, UTF-32BE, UTF-32LE

Windows Encoding

  • Windows-1250 for Central European languages that use Latin script, (Polish, Czech, Slovak, Hungarian, Slovene, Croatian, Romanian and Albanian)
  • Windows-1251 for Cyrillic alphabets
  • Windows-1252 for Western languages
  • Windows-1253 for Greek
  • Windows-1254 for Turkish
  • Windows-1255 for Hebrew
  • Windows-1256 for Arabic
  • Windows-1257 for baltic languages
  • Windows-1258 for Vietnamese
  • Windows-31J for Japanese
