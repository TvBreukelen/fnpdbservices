Export From

Default Settings

  • Combobox (1)
    • In the first combobox you can select the format of the "Export From" file (Calc, JFile, JSON, YAML,.. etc.) that you'd like to export.

  • Combobox (2)
    • In the second combobox you can select a file or host database from the list of previously defined "Export From" files of the same kind.

  • Open
    • You can use the Open button to select the file to be exported.

  • Table / Worksheet
    • If your database or spreadsheet has more than one table or worksheet then you can select in this comboxbox the table/worksheet to export.

Export From TextFile

  • Export Format
    • The export format specifies the the textfile format you are exporting from.

    • Standard CSV File
      • Export from a standard CSV File, created for example by HanDBase, JFile, MS-Excel or other program.

    • Other
      • Exports from a delimited text file according to your own requirements.

  • Other
    • The right part of the screen contains the options for exporting from another textfile format It contains of the following parts:
    • Field Separator
      • By selecting the field separator you can either export from a Comma-, Tab-, Semi colon- or pipe delimited file
    • Text Delimiter
      • This specifies the text delimiter (to enclose all database fields).

    Important: DBConvert checks whether the database file truly exists and whether its format corresponds with the selected database format. DBConvert is not able to read encrypted databases. When DBConvert opens a DBase or FoxPro database it tries to find the encoding that was used to create the database. If not found, DBConvert will use the standard encoding.

    Dbase or FoxPro Memo fields that contain binary data (like OLE-Objects or pictures) are not supported, DBConvert can only read memo texts. DBConvert can read Visual FoxPro files, but doesn't support the non-standard xBase fieldtypes, like Currency, DateTime or Variant.

    Export To

    Default Settings

    • Combobox
      • In the combobox you can select the format of the "Export To" file (Calc, HanDBase, JFile, ... etc.).

    • Filename / Open
      • You can use the Open button to select the database to export to or type in the name of the database directly in the text field.

      Database / Worksheet This field identifies the database or worksheet name as it should appear in your export file. For spreadsheets it represents the tab sheet label and for XML files the root- element. If left unchanged, DBConvert uses this as default database name for the export file. Should your export file require another name then you could enter the new name in this field.

    • Skip empty records
      • Sometimes it could occur that all fields that you've selected are empty in a record. By default these records are imported. If you don't want to have them included you can activate this check box.

    • Create Backup
      • Should the export file already exist then this option allows you to create a backup copy of the previous export file. If you deactivate the option then the previous copy of the export file will be overwritten.

    Notes

    DBConvert verifies whether the export file exists and whether its format corresponds with the selected database format. FNProg2PDA is not able to write encrypted databases
