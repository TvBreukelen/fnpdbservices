SSH tunnel

Use the fields in the SSH Tunnel tab to configure SSH Tunneling. You can use the "SSH Tunnel" tab to connect DBConvert (through an intermediary proxy host) to a server that resides on a network to which the client may not be able to connect directly.

Use SSH tunnel

If you don't need to use a SSH tunnel then leave this check box deactivated. If SSH is required you you'd need to activate it.

SSH Host

Here you'd specify the name or IP address of the SSH host (through which client connections will be forwarded).

Port

Enter here port of the SSH Host through which client connections will be forwarded). The default port number is 22.

Username

In the User name field you'd specify the name of a user with login privileges for the SSH host.

Password

If the SSH host is expecting a password for the user name or private key file (if being used), you can enter here the password. FYI: by default a private key file password is assumed.

Private key file

If the SSH host is expecting a private key file for authentication, use this field to specify the location of the key file. This should be an OpenSSH file. FYI: the most recent private key files created by PuTTY are incompatible. However, with the PuTTYGen program you can convert them to the correct OpenSSH format.
